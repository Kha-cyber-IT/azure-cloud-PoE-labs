# Day 17: Create a Public Azure Blob Storage Container

Create a new storage account named `xfusionst12898` and a `public` Blob container named `xfusion-blob-26174` within the storage account. Make sure `anonymous read access` for containers and blobs is enabled.

### Create a new storage account named `xfusionst12898`

1. Go to `Storage center` and click `+ Create`.

   <img width="1194" height="523" alt="image" src="https://github.com/user-attachments/assets/ee00865d-29ff-4375-94e0-b40760da20b5" />

2. Create storage account with preffered name and other details.

   <img width="935" height="568" alt="image" src="https://github.com/user-attachments/assets/d3dedca7-140d-4e34-b2e6-a398fcb4f949" />

4. Under `Review + create` tab, preview details and click `Create`.

   <img width="934" height="572" alt="image" src="https://github.com/user-attachments/assets/851bdfc4-cf30-4a32-923a-053e29a51906" />

5. Go to resource and in the navigation pane, click on `Storage browser`.

   <img width="1302" height="572" alt="image" src="https://github.com/user-attachments/assets/1044e2e5-b516-4852-b4d4-9e361477c775" />

6. To create public blob container, you have to enable annonymous access from `Navigation pane, Settings, Configuration` and `Save`.

   <img width="934" height="529" alt="image" src="https://github.com/user-attachments/assets/7f55e439-9e40-4204-93af-94dcde531f5b" />

6. Select `Blob container` and click on `+ Add container`.

   <img width="1098" height="305" alt="image" src="https://github.com/user-attachments/assets/50d3fb05-ec0d-43f4-b359-c81ed536e867" />

7. Enter blob name and click `Create`

   <img width="435" height="570" alt="image" src="https://github.com/user-attachments/assets/77cdc199-167c-4031-b4ec-55a9a66c18c7" />
